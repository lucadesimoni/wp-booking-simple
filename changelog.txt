*** WP booking Luca Changelog ***

2026.06.12 - version 1.7.0
 * New - Admin calendar overview to see all bookings at a glance: month and list views, entries colour-coded by status (pending/confirmed/cancelled) with party size, summary cards (upcoming, pending, confirmed, total), a legend, "next check-in" note, and the calendar opens on the next upcoming booking.
 * New - "Owner" column on the All Bookings table; Owner and "Visitors welcome?" are also shown in the booking details popup.
 * Tweak - Flatpickr (date picker) and FullCalendar are now bundled with the plugin instead of loaded from a CDN, so the booking form and calendar keep working behind a strict Content-Security-Policy, on intranets, or when the CDN is unreachable.

2026.06.12 - version 1.6.0
 * New - Drag-and-drop email template builder: arrange Text, Heading, Booking-details, Button, Image and Divider blocks for each email, reorder them by dragging, and insert merge tags from a dropdown. Blocks take precedence over the plain-text body, which remains as a fallback.
 * New - Configurable booking-form fields: an "Owner" dropdown populated from a list of names you define in Settings, and a "Visitors welcome?" yes/no field. Both are stored with the booking and exposed as the {owner} and {visitors_welcome} merge tags.
 * Tweak - The bookings table now stores owner and visitors_welcome; the columns are added automatically on upgrade.
 * Fix - Admin styles/scripts now load on the Settings and All Bookings screens (the hook match was too narrow).

2026.06.12 - version 1.5.0
 * New - Booking confirmation and admin notification emails now include an iCalendar (.ics) attachment named booking-{id}.ics, so guests and owners can add the stay to their calendar in one tap. The all-day event spans check-in to check-out.
 * Tweak - Email markup hardened for dark-mode clients (Outlook/Gmail): forced light color-scheme via meta tags and explicit inline colors on the header, content, details box and buttons.

2026.06.12 - version 1.4.0
 * New - Customizable email templates: edit the subject and body of the guest confirmation, guest cancellation and admin notification emails from Settings > Email Templates.
 * New - Merge tags ({guest_name}, {check_in}, {check_out}, {guests}, {total_price}, {booking_details}, {manage_link}, {admin_link}, and more) are replaced with each booking's details; clearing a field restores the built-in default.
 * Tweak - The branded HTML email layout is preserved around custom content, and guest-supplied values are escaped to prevent markup injection.

2026.06.12 - version 1.3.0
 * New - Built-in SMTP email delivery: send booking confirmations and notifications through an external mailbox such as Gmail / Google Workspace (host, port, encryption, authentication, username and app password) for reliable delivery.
 * New - "Send Test Email" button in Settings to verify delivery, with success/error feedback (captures and shows SMTP errors).
 * New - The configured "From" address and name are now applied to all site mail via wp_mail_from / wp_mail_from_name.
 * New - Contextual admin reminder to enable SMTP, shown only on the plugin's own screens while SMTP is disabled.

2026.06.11 - version 1.2.0
 * New - Configurable booking entry options in Settings: minimum/maximum stay (nights), minimum advance notice and booking window (days ahead), default adults/kids on the form, require phone number, show/hide the notes field, and auto-confirm new bookings.
 * New - Stay-length and booking-window rules are enforced both in the date picker and server-side; the date picker now limits selectable dates accordingly.

2026.06.11 - version 1.1.1
 * New - The booking form and calendar can be embedded on ANY existing page (including page builders like Elementor/Divi/WPBakery); required CSS/JS now load automatically wherever the shortcode, block or widget renders.

2026.06.11 - version 1.1.0
 * Fix - Plugin no longer fatally errors on activation (class files now match the loader; removed a duplicate class).
 * Fix - Stripped UTF-8 BOM from all PHP/JS files that caused "headers already sent" errors.
 * Fix - Frontend calendar/widget now read the correct localized data object, so availability loads.
 * New - Activation auto-creates a "Book Now" page and a "Manage Booking" page so magic links and embedding work out of the box.
 * New - Email magic link resolves to the real management page permalink instead of a hardcoded slug.
 * New - Gutenberg "Booking Form" block (in addition to the calendar block).
 * New - Admins can Confirm and Cancel bookings from the bookings list (with guest notification on cancel).
 * Tweak - Booking assets load only on pages that use the form, calendar or widget (faster site).
 * Tweak - Extracted pricing/validation into a testable helpers class.
 * Tweak - Added a standalone test runner and a WordPress PHPUnit suite, plus a clean build script.

2025.11.28 - version 1.0.0
 * Initial release
 * Admin calendar interface
 * Frontend booking form
 * Price calculation based on adults and kids
 * Email notifications
 * Booking management and cancellation system
