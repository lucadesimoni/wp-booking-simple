*** WP booking Luca Changelog ***

2026.06.16 - version 1.18.1
 * New - "Refunded" payment status, completing the payment lifecycle: mark a booking refunded from the editor (set the amount paid back to 0); it shows a Refunded badge on the bookings list, is recorded in the change history, and the TWINT QR code never shows for a refunded or cancelled booking.
 * Tweak - Verified end to end: the guest's TWINT QR tracks the outstanding balance (e.g. 500 → 300 after a deposit) and disappears once the booking is paid in full, cancelled or refunded.

2026.06.16 - version 1.18.0
 * New - TWINT / Swiss QR-bill payment integration (free, no merchant account). When enabled, a booking with an outstanding balance shows a Swiss QR code on the guest's manage page; the guest scans it with TWINT or any Swiss banking app to pay. Configure your IBAN and address under Settings > General > TWINT / QR-bill Payments. The payload follows Swiss Payment Standards QR-bill 2.0 and the QR generator (MIT, Kazuhiko Arase) is bundled with the plugin — nothing is sent to an external service.

2026.06.16 - version 1.17.0
 * New - Tooltips on the availability calendar days: hovering shows whether a date is available ("click to select your dates"), already booked, or in the past. Translated into German.
 * Tweak - Past dates are now clearly deactivated on the calendar (muted background, not-allowed cursor, not selectable) to match the date picker, which already blocked them.

2026.06.16 - version 1.16.0
 * New - Colour controls for the Gutenberg blocks: a Colors panel on the "Booking Form" (accent, button background/text/hover) and "Booking Calendar" (accent, booked) blocks, applied per block instance via uniquely-scoped CSS (with strict colour validation). Non-Elementor users now get the same styling options.
 * New - Sizing controls on the Elementor widgets: button font size and corner radius (form) and day-cell height (calendar).

2026.06.16 - version 1.15.1
 * New - The Elementor "Booking Form" and "Booking Calendar" widgets gained a Style tab: pick the accent colour and button colours (form), or the accent and booked colours (calendar), directly from Elementor's design panel. The colours are applied with per-widget scoped CSS so each instance can differ.

2026.06.16 - version 1.15.0
 * New - Native Elementor widgets: "Booking Form" and "Booking Calendar" now appear in the Elementor editor under a dedicated "WP booking Luca" category, each with a Title control. They render the same form/calendar as the shortcodes.
 * New - The Gutenberg blocks now live in their own discoverable "WP booking Luca" category (instead of the generic Widgets group) and carry full server-side metadata. This also makes them easy to find in Spectra, which uses the standard block inserter.

2026.06.15 - version 1.14.0
 * New - Theme compatibility hardening for Astra / Astra Pro and other opinionated themes. The booking form, primary/cancel/secondary buttons, dropdowns and the availability calendar are re-asserted with wrapper-scoped CSS so theme globals (e.g. `.ast-container button`, `.entry-content input/select` resets, forced link underlines) can no longer override or break the booking UI, while the calendar's click-to-select date picking keeps working.
 * Note - Verified against a faithful simulation of Astra's signature global styles (container-scoped button/input resets, forced link underlines, 15px base, box-sizing): the submit button stays branded red, inputs/selects stay white at a consistent 48px height, calendar links stay un-underlined, and range selection still functions.

2026.06.15 - version 1.13.2
 * Tweak - The frontend availability calendar, the calendar widget and the date picker now start the week on Monday (the admin calendar already did).

2026.06.15 - version 1.13.1
 * Tweak - Refreshed the availability calendar styling: no more underlined day numbers / weekday headers, lighter grid lines, clearer selected-range highlight, and compact cells that fit on both desktop and mobile. (Fixed selectors that never applied because FullCalendar shares the `.fc` class with the calendar container.)

2026.06.15 - version 1.13.0
 * New - Select your stay directly on the availability calendar: click check-in, then check-out, and the range is highlighted and pushed into the booking form with the price recalculated. Past/booked days are unselectable and you cannot span an existing booking.
 * Tweak - Booked dates are disabled in the date picker (previously only rejected on submit). The calendar's selection logic moved into the static frontend script, fixing a bug where WordPress's content filter corrupted the inline script so calendar clicks never reached the form.
 * Tweak - Form-control consistency: the Owner and "Visitors welcome?" dropdowns now match the inputs' height, with unified borders/padding and theme-inherited fonts.

2026.06.14 - version 1.12.0
 * Tweak - Polished the booking confirmation: on success the form is now replaced by a clear confirmation panel (checkmark, the success message, a reassuring "check your email / spam folder" note, and a "Book another stay" button) instead of leaving an empty form on screen. New strings are translated into German.

2026.06.14 - version 1.11.0
 * Tweak - Restructured the Settings screen into clear tabbed sections (General, Booking Rules, Booking Form, Email Delivery, Email Templates) for much easier navigation. It remains a single form with one Save button, and the tab you saved on stays open afterwards.
 * Tweak - The "Send a Test Email" tool now lives under the Email Delivery tab, alongside the SMTP settings.
 * i18n - Translated the new settings strings (General, Email Delivery, Pricing & Notifications) into German.

2026.06.14 - version 1.10.0
 * New - Full German translation (de_DE and de_CH) for the admin, booking form and emails. Set the site language to German to use it; the .po/.mo and a .pot template ship under /lang, with a tools/i18n/po2mo.php compiler for editing translations.
 * Tweak - Responsive admin: the bookings table now scrolls horizontally on small screens instead of overflowing the page; the booking editor and dashboard collapse to a single column, and the stat cards, payment fields and dashboard filter wrap cleanly on phones.

2026.06.12 - version 1.9.0
 * New - CSV export: download all bookings with full payment details (amount paid, outstanding balance, status, method) from the All Bookings and Dashboard screens.
 * New - Dashboard date-range filter (by check-in date); KPIs, breakdowns and the CSV export all respect the active range.
 * New - Payment-reminder email: send a reminder to the guest from the booking editor. The template (subject, body and visual blocks) is editable under Settings > Email Templates.
 * New - {amount_due} merge tag for the outstanding balance. The {payment_status}, {payment_method}, {amount_paid} and {amount_due} tags are now listed in the email builder's tag palette.

2026.06.12 - version 1.8.0
 * New - Per-booking payment tracking: payment status (Unpaid / Partially paid / Paid), payment method (Bank, TWINT, Cash) and the amount effectively paid. A Payment column on the bookings list shows status and paid/total at a glance.
 * New - Full booking editor in a modal: edit any field (guest details, dates, number of guests, owner, price, status, payment, notes). Availability is re-checked to prevent overlaps and a "Recalc" button recomputes the price from the dates and guests.
 * New - Change history per booking: every later edit (and status change) is recorded with who made it, when, and the before/after of each field, shown as a timeline in the editor.
 * New - Insights dashboard (WP booking Luca > Dashboard): KPIs for bookings, nights, guests, revenue, collected and outstanding amounts, plus bookings per guest, owner usage, payments by method and bookings per month.
 * New - {payment_status}, {payment_method} and {amount_paid} email merge tags.
 * Tweak - Adds the payment columns and a booking-history table automatically on upgrade.

2026.06.12 - version 1.7.0
 * New - Admin calendar overview to see all bookings at a glance: month and list views, entries colour-coded by status (pending/confirmed/cancelled) with party size, summary cards (upcoming, pending, confirmed, total), a legend, "next check-in" note, and the calendar opens on the next upcoming booking.
 * New - "Owner" column on the All Bookings table; Owner and "Visitors welcome?" are also shown in the booking details popup.
 * Tweak - Flatpickr (date picker) and FullCalendar are now bundled with the plugin instead of loaded from a CDN, so the booking form and calendar keep working behind a strict Content-Security-Policy, on intranets, or when the CDN is unreachable.

2026.06.12 - version 1.6.0
 * New - Drag-and-drop email template builder: arrange Text, Heading, Booking-details, Button, Image and Divider blocks for each email, reorder them by dragging, and insert merge tags from a dropdown. Blocks take precedence over the plain-text body, which remains as a fallback.
 * New - Configurable booking-form fields: an "Owner" dropdown populated from a list of names you define in Settings, and a "Visitors welcome?" yes/no field. Both are stored with the booking and exposed as the {owner} and {visitors_welcome} merge tags.
 * Tweak - The bookings table now stores owner and visitors_welcome; the columns are added automatically on upgrade.
 * Fix - Admin styles/scripts now load on the Settings and All Bookings screens (the hook match was too narrow).

2026.06.12 - version 1.5.0
 * New - Booking confirmation and admin notification emails now include an iCalendar (.ics) attachment named booking-{id}.ics, so guests and owners can add the stay to their calendar in one tap. The all-day event spans check-in to check-out.
 * Tweak - Email markup hardened for dark-mode clients (Outlook/Gmail): forced light color-scheme via meta tags and explicit inline colors on the header, content, details box and buttons.

2026.06.12 - version 1.4.0
 * New - Customizable email templates: edit the subject and body of the guest confirmation, guest cancellation and admin notification emails from Settings > Email Templates.
 * New - Merge tags ({guest_name}, {check_in}, {check_out}, {guests}, {total_price}, {booking_details}, {manage_link}, {admin_link}, and more) are replaced with each booking's details; clearing a field restores the built-in default.
 * Tweak - The branded HTML email layout is preserved around custom content, and guest-supplied values are escaped to prevent markup injection.

2026.06.12 - version 1.3.0
 * New - Built-in SMTP email delivery: send booking confirmations and notifications through an external mailbox such as Gmail / Google Workspace (host, port, encryption, authentication, username and app password) for reliable delivery.
 * New - "Send Test Email" button in Settings to verify delivery, with success/error feedback (captures and shows SMTP errors).
 * New - The configured "From" address and name are now applied to all site mail via wp_mail_from / wp_mail_from_name.
 * New - Contextual admin reminder to enable SMTP, shown only on the plugin's own screens while SMTP is disabled.

2026.06.11 - version 1.2.0

 * New - Stay-length and booking-window rules are enforced both in the date picker and server-side; the date picker now limits selectable dates accordingly.

2026.06.11 - version 1.1.1
